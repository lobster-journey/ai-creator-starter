# 龙虾巡游记公司 — 组织架构

> 版本：v1.0 | 制定人：总经理🦞 | 日期：2026-04-22

## 公司定位
AI智能体全自主运营的科技内容公司。
核心业务：内容创作、公司调研、品牌运营。
核心理念：设计按全自动化，执行按当前合规边界。

## 组织架构

### 总经办 HQ

|------|-----------|------|

| 角色 | Agent标识 | 职责 |
|------|-----------|------|
| 热点侦察员🔍 | `agent-hot-scout` | 每日搜索AI/科技热点Top10，评估热度和传播潜力，输出选题候选报告 |
| 文案创作员✍️ | `agent-copywriter` | 接收选题，创作完整小红书笔记（标题+正文+标签），风格接地气有干货 |

## 协作流程（含上下游依赖）

### 08:00 每日内容生产

```
【阶段1 串行】
总经理 → spawn 热点侦察员
           ↓ 等待完成，拿到选题报告
【阶段2 并发】
总经理 → spawn 文案创作员（依赖选题报告）
       → spawn 配图师（依赖选题关键词，与文案并发）
           ↓ 两者都完成后
【阶段3 串行】
总经理 → spawn 质检员（依赖完整笔记文案）
           ↓ 通过则交付，不通过返工（最多5次）
【交付】
总经理汇总：笔记文案 + 3张配图 → 发给用户
```

> 关键依赖：热点侦察员 → 文案创作员 → 质检员（串行）
> 唯一并发：文案创作员 ∥ 配图师（同时依赖选题，互不干扰）

### 21:30 每日公司调研

```
【阶段1 并发】
总经理 → spawn 调研员1（公司A）
       → spawn 调研员2（公司B）  ← 5人完全并发，互相独立
       → spawn 调研员3（公司C）
       → spawn 调研员4（公司D）
       → spawn 调研员5（公司E）
           ↓ 等待全部完成（或各自超时）
【阶段2 串行】
总经理 → spawn 洞察提炼师（依赖5篇调研报告）
           ↓
【交付】
总经理推送：5篇报告 + 共性洞察 → 发给用户
```

> 关键依赖：调研员1~5（并发）→ 洞察提炼师（串行等待）

### 22:00 每日工作复盘
```
总经理 → spawn 复盘官 → 生成图文日报 → 发送
```

## 核心工作原则

### ⚡ 调研与深度思考必须基于实时信息

**所有调研类、分析类、热点类工作，严禁直接基于已有知识生成内容。**
必须通过以下方式获取实时信息后再进行分析：

| 信息来源 | 适用场景 | 工具 |
|----------|----------|------|
| 搜索引擎 | 最新动态、发布新闻、数据 | querit-search / baidu-search / google-search |
| 微博 | 热搜话题、实时舆情、社会热点 | 搜索「微博热搜」或通过浏览器访问 |
| 小红书 | 内容趋势、博主动态、用户反馈 | 浏览器访问 |
| 知乎 | 深度讨论、行业洞察 | 搜索或浏览器 |
| GitHub | 开源项目、技术动态 | web_fetch / 搜索 |
| 官网 / 新闻 | 产品发布、公司公告 | web_fetch |

**违反此原则 = 工作不合格，必须返工。**

---

## 执行规则

1. **总经理不亲自执行具体任务**，只负责调度和汇总
2. **每个子Agent有明确的输入输出格式**，不做超出职责的事
3. **子Agent超时15分钟自动终止**，主Agent记录失败并继续
4. **质检不通过最多返工5次**，第6次直接选最优版本交付
5. **所有结果汇总后由总经理发送给用户**，子Agent不直接对外发消息
