# 公司调研工作流 Skill

## 功能描述
系统化调研目标公司，生成结构化报告，提炼可借鉴经验。

## 工作流程

### 步骤1：目标筛选（15分钟）
**目标**：确定调研目标清单
**方法**：
- 搜索"一人公司"、"独立开发者"、"indie hacker"等关键词
- 按热度（GitHub Stars、社区讨论、媒体报道）排序
- 优先选择科技领域、有公开信息的公司
- 排除信息过少或非技术类公司

**产出**：
- 目标公司清单（100家）
- 每家公司包含：名称、GitHub地址、官网、热度评分

**验收标准**：
- 清单包含100家公司
- 每家公司至少有GitHub或官网信息
- 热度评分合理分布

---

### 步骤2：信息采集（每家30分钟，共50小时）
**目标**：全面采集目标公司信息

**信息维度**：
1. **基本信息**
   - 公司/项目名称
   - 创始人背景
   - 成立时间
   - 团队规模
   - 所在地

2. **产品/服务**
   - 核心产品
   - 解决的问题
   - 目标用户
   - 产品形态（开源项目/SaaS/工具/平台）

3. **技术架构**
   - 技术栈
   - 开源/闭源
   - 技术亮点
   - 代码质量（如有）

4. **商业模式**
   - 收费方式
   - 定价策略
   - 目标营收（如有）
   - 融资情况

5. **发展历程**
   - 重要里程碑
   - 增长曲线
   - 用户规模变化

6. **社区影响力**
   - GitHub Stars/Forks
   - 社区活跃度
   - 媒体报道
   - 用户评价

7. **成功要素**
   - 核心竞争力
   - 关键决策
   - 成功原因分析

8. **潜在风险**
   - 商业风险
   - 技术风险
   - 市场风险

**方法**：
- GitHub主页分析
- 官网内容抓取
- 社交媒体搜索
- 新闻报道查询
- 用户评价收集

**产出**：
- 每家公司一份调研报告（500-1000字）
- 结构化数据（JSON格式）

**验收标准**：
- 每份报告包含所有8个维度
- 信息准确可验证
- 分析客观深入

---

**报告模板**：

- 产品名称：
- 产品描述：
- 解决的问题：
- 目标用户：

## 3. 技术架构
- 技术栈：
- 开源/闭源：
- 技术亮点：

## 4. 商业模式
- 收费方式：
- 定价策略：
- 营收情况：

## 5. 发展历程
- 重要里程碑：
- 增长曲线：
- 用户规模：

## 6. 社区影响力
- GitHub Stars：
- 社区活跃度：
- 媒体报道：

## 7. 成功要素
- 核心竞争力：
- 关键决策：
- 成功原因：

## 8. 可借鉴点

- 商业风险：
- 技术风险：

- 100份结构化报告
- 保存为Markdown文件

**验收标准**：
- 每份报告结构完整
- 分析深入有见地
- 可借鉴点具体可行

---

### 步骤4：数据汇总（3小时）
**目标**：汇总所有调研数据，发现规律

**汇总维度**：
1. **行业分布**
   - 不同领域公司数量
   - 热门赛道分析

2. **技术栈统计**
   - 常用技术栈排名
   - 技术选型趋势

3. **商业模式分析**
   - 主流收费方式
   - 定价策略对比

4. **成功要素归纳**
   - 高频成功因素
   - 关键决策模式

5. **增长策略总结**
   - 有效增长手段
   - 增长曲线类型

6. **风险识别**
   - 常见风险类型
   - 风险应对策略

**产出**：
- 数据汇总报告（3000-5000字）
- 可视化图表（可选）

**验收标准**：
- 数据统计准确
- 规律发现有价值
- 图表清晰易懂

---

### 步骤5：建议提炼（2小时）
**目标**：基于调研结果，提出具体建议

**建议维度**：
1. **产品方向**
   - 产品定位建议
   - 功能优先级
   - 差异化策略

2. **技术选型**
   - 技术栈建议
   - 架构设计参考
   - 开源策略

3. **商业模式**
   - 收费方式建议
   - 定价策略
   - 变现路径

4. **增长策略**
   - 用户获取方法
   - 社区运营策略
   - 品牌建设思路

5. **风险管理**
   - 风险预防措施
   - 应急预案
   - 合规建议

**产出**：
- 50+条具体建议（每条100-200字）
- 建议优先级排序
- 实施路径建议

**验收标准**：
- 建议具体可行
- 有成功案例支撑
- 优先级合理

---

### 步骤6：报告整理（1小时）
**目标**：整理最终产出，便于查阅

**产出文件**：
1. **调研总报告**（RESEARCH_REPORT.md）
   - 调研背景和方法
   - 数据汇总分析
   - 50+条建议
   - 附录（100家公司清单）

2. **单篇报告**（reports/目录）
   - 100份独立报告
   - 按热度排序命名

3. **数据汇总表**（DATA_SUMMARY.csv）
   - 公司名称
   - GitHub地址
   - 官网
   - Stars数
   - 商业模式
   - 核心产品
   - 成功要素

**验收标准**：
- 文件结构清晰
- 内容完整准确
- 便于查阅检索

## 注意事项

1. **信息真实性**
   - 优先使用官方信息源
   - 交叉验证关键数据
   - 标注信息来源

2. **分析客观性**
   - 避免主观臆断
   - 基于数据说话
   - 多角度分析

3. **建议可行性**
   - 结合自身情况
   - 考虑资源限制
   - 提供实施路径

4. **时间效率**
   - 并行采集信息
   - 使用自动化工具
   - 定期休息避免疲劳
