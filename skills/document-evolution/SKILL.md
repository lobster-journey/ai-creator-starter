# 文档阅读与进化Skill

**核心目标**：通过阅读文档持续进化公司经营策略、技术方案和代码实现。

---

## 核心能力

### 1️⃣ 文档阅读 ✅
- 微信公众号文章
- 技术博客
- 官方文档
- 行业报告

### 2️⃣ 信息提炼 ✅
- 核心观点提取
- 关键技术点总结
- 最佳实践归纳
- 案例分析

### 3️⃣ 业务分析 ✅
- 对业务的帮助
- 可借鉴的经验
- 优化建议
- 实施路径

### 4️⃣ 技术分析 ✅
- 技术架构优化
- 代码模式改进
- 工具链升级
- 性能提升方案

### 5️⃣ 代码改进 ✅
- 代码库修改
- 测试验证
- 提交代码
- 文档更新

---

## 使用流程

### 标准流程

```
文档阅读 → 信息提炼 → 业务分析 → 技术分析 → 代码改进 → 测试验证 → 提交代码
```

### 详细步骤

**步骤1：阅读文档**
- 使用web_fetch或wechat-article-to-markdown获取文档
- 提取完整内容
- 保存为Markdown

**步骤2：提炼信息**
- 总结核心观点
- 提取关键技术
- 记录最佳实践

**步骤3：业务分析**
- 分析对业务的帮助
- 提出优化建议
- 制定实施计划

**步骤4：技术分析**
- 评估技术架构
- 识别改进点
- 设计优化方案

**步骤5：代码改进**
- 修改代码库
- 编写测试用例
- 验证功能
- 提交代码

---

## 代码修改规则

### ⚠️ 重要原则

**所有代码修改必须**：
1. ✅ 先分析，再设计，最后实现
2. ✅ 编写测试用例
3. ✅ 本地测试通过
4. ✅ 验证功能正常
5. ✅ 更新相关文档
6. ✅ 提交到Git

### 测试验证流程

```bash
# 1. 修改代码
vim path/to/file.py

# 2. 编写测试
vim tests/test_file.py

# 3. 运行测试
pytest tests/test_file.py -v

# 4. 验证功能
python3 -m path/to/file.py

# 5. 提交代码
git add .
git commit -m "feat: 优化xxx"
git push
```
